# 更新验收 Checklist

> 起草：质检员 @ 2026-05-05
> 更新：程序员先生 @ 2026-05-05
> 状态：草案
> 用途：用于已有验收结论的项目，在发生改动后做增量复核

## 一、前置确认

- [ ] 这个项目已有过历史验收结论
- [ ] 已找到上次验收报告 / 结论
- [ ] 已明确本次是更新验收，不是从零开始的新项目验收

## 二、变更范围确认

- [ ] 改了哪些文件
- [ ] 改的是文档 / 配置 / 功能 / 测试 / 部署 / 依赖 / 数据结构 / 权限逻辑
- [ ] 改动是否影响核心路径
- [ ] 改动是否影响历史结论中的问题项
- [ ] 改动是否引入新的安全 / 隐私 / 高权限 / 外发风险

## 三、更新分级与复核深度

- [ ] 已按 `tiered-review-standard.md` 的**变更类型 → 复核深度表**判断本次级别
- [ ] 已确认本次是轻量更新验收 / 标准更新验收 / 局部深验 / 完整深验

## 四、执行动作

- [ ] 已检查本次改动范围
- [ ] 已跑相关 build / test / CI / 手测
- [ ] 已复查受影响旧功能
- [ ] 若命中高风险变更，已执行局部深验或完整深验

## 五、旧结论是否失效

- [ ] 上次通过项是否仍成立
- [ ] 上次问题项是否已修复
- [ ] 上次风险判断是否仍成立
- [ ] 是否出现让旧结论部分失效的新改动

## 六、更新验收输出

- [ ] 已写清本次更新范围
- [ ] 已写清本次复核深度
- [ ] 已写清哪些旧结论仍有效
- [ ] 已写清哪些旧结论失效
- [ ] 已写清是否需要重新深度验收
- [ ] 已给出本次更新后的新结论
